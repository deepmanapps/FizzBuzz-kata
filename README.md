# FizzBuzz Kata

This project is an implementation of the FizzBuzz kata.

## Test-Driven Development (TDD)

This project was developed using TDD. The benefits of using TDD are:

*   **Better Design:** Writing tests first forces you to think about the design of the code from a user's perspective.
*   **Higher Quality:** TDD leads to higher quality code because you have a comprehensive suite of tests that can be run at any time.
*   **Reduced Bugs:** With TDD, you are less likely to have bugs in your code because you are constantly testing it.
*   **Confidence to Refactor:** Having a good set of tests gives you the confidence to refactor your code without fear of breaking it.

## SOLID Principles

The SOLID principles are a set of five design principles for object-oriented software development. The "O" in SOLID stands for the **Open/Closed Principle**.

### Open/Closed Principle

The Open/Closed Principle states that software entities (classes, modules, functions, etc.) should be open for extension, but closed for modification. In the context of this FizzBuzz kata, this means that we should be able to add new rules (e.g., for numbers divisible by 7) without modifying the existing code. This can be achieved by using interfaces and dependency injection.
